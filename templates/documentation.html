Alexa Sentiment Analysis Documentation

A machine learning-powered web application that analyzes Amazon Alexa reviews to determine sentiment polarity (positive/negative).

Key Features

  • Real-time sentiment analysis of Alexa reviews
  • User authentication system
  • Profile management
  • Contact form functionality
  • Responsive design with dark mode support
  • Interactive UI elements

Technology Stack

Backend

  • Python Flask framework
  • SQLite database
  • NLTK for text processing
  • Scikit-learn for ML model

Frontend

  • HTML5, CSS3, JavaScript
  • Responsive design
  • Interactive animations

Machine Learning Model

Data Preprocessing

  • HTML tag removal
  • Text tokenization
  • Stop words removal
  • Porter Stemming

Model Architecture

  • Algorithm: Support Vector Classification (SVC)
  • Feature Extraction: CountVectorizer
  • Training Split: 70-30

Database Schema

Users Table


CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT 1
)

Contacts Table


CREATE TABLE contacts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT NOT NULL,
    subject TEXT NOT NULL,
    message TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status TEXT DEFAULT 'unread'
)

Security Features

  • Password hashing using Werkzeug
  • Session management
  • CSRF protection
  • SQL injection prevention
  • Input validation

Usage Guide

Sentiment Analysis

  1. Enter your Alexa review in the text area
  2. Click "Analyze Sentiment" button
  3. View the sentiment prediction (Positive/Negative)

User Features

  • Create an account using email
  • Login to access additional features
  • Update profile information
  • Change password
  • Contact support team
